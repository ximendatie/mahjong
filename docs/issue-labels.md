# Issue Labels

Use these labels to keep contributor work easy to browse.

| Label | Use for |
| --- | --- |
| `provider` | New or changed local provider support. |
| `parser` | Local metadata parsing, schema drift, fixture coverage. |
| `privacy` | Data boundaries, permissions, redaction, sensitive display behavior. |
| `ui` | Board, settings, menu bar, desktop pet, and visual polish. |
| `diagnostics` | Provider Diagnostics status, copy, paths, and refresh feedback. |
| `docs` | README, architecture, privacy, contributing, and provider docs. |
| `tests` | Unit tests, fixtures, CI reliability, test helpers. |
| `release` | Packaging, signing, notarization, release notes, archives. |
| `good first issue` | Small, well-scoped tasks with clear files and acceptance criteria. |
| `help wanted` | Maintainer-approved tasks where outside contributions are welcome. |
